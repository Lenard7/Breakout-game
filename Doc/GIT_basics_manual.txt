to rename a file and note it as git change:
$ git mv old_filename new_filename
to move a file to new location, there is no special procedure, just move file to other location and commit changes


A file should be ignored if any of the following is true:
	-1- The file is not used by your project
	-2- The file is not used by anyone else in your team
	-3- The file is generated by another process
	
gitignore file should be allocated in root folder of git repository (recommended) (it can be placed on other locations of git repository as well)
generating .gitignore file:
1) On Windows:
	-1- Create the text file gitignore.txt
	-2- Open it in a text editor and add your rules, then save and close
	-3- Hold SHIFT, right click the folder you're in, then select Open command window here
	-4- Then rename the file in the command line, with ren gitignore.txt .gitignore
	
2) On Linux:
	-1- touch .gitignore
